# Проверка и настройка **Журнала записи**
Для автотестирования созданы фича-сценарии, охватывающие:
* загрузку тестовых данных,
* проверку нового журнала записи,
* создание и обработку приёма,
* проверку старого журнала записи.

## **1️⃣ Общий контекст и подготовка**
Во всех сценариях используется единый контекст:
* Запуск или подключение к уже открытому `TestClient`.
* Закрытие всех открытых окон клиентского приложения.
* Очистка переменных перед началом сценариев (где требуется).


## **2️⃣ Сценарий «Загрузка данных»**
### **Цель:**
Подготовить тестовое окружение для работы с журналом записи.

### **Выполняемые действия:**
#### 🔹 Загрузка справочных данных
* Очистка всех переменных.
* Создание:
  * услуги,
  * кабинета,
  * врача,
  * пациента.
* Формирование дат для дальнейшей записи.

#### 🔹 Создание графика работы врача
* Переход в карточку врача.
* Открытие формы **«График работы»**.
* Запуск мастера заполнения графика.
* Установка режима **«Циклически»**.
* Добавление временного интервала:
  * начало — `00:30`,
  * окончание — `23:30`.
* Сохранение и формирование графика.
* Возврат в основную форму и закрытие карточки врача.

## **3️⃣ Сценарий «Проверка фильтров (кнопки)»**
### **Цель:**
Проверить работу элементов управления и настроек нового журнала записи.

### **Основные проверки:**
#### 🔹 Навигация по журналу
* Переход в **Журнал записи**.
* Проверка перехода в новый журнал.
* Работа стрелок переключения периода (влево / вправо).
* Открытие и закрытие выбора даты.

#### 🔹 Фильтры по времени
* Переключение фильтров:
  * День,
  * Утро,
  * Вечер,
  * Весь день.

#### 🔹 Дополнительные фильтры
* Фильтр по должности.
* Фильтры по статусам, меткам, специализациям.
* Очистка всех фильтров.

#### 🔹 Проверка печати
* Открытие формы печати.
* Закрытие формы без ошибок.

## **4️⃣ Проверка настроек журнала записи**
### **Настройки заголовка записи**

* Переключение вариантов отображения:
  * не отображать,
  * фамилия И.О. пациента,
  * время записи.
* Проверка дополнительной информации в заголовке:
  * кабинет / врач,
  * ассистент.

### **Настройки отображения записи**
* Проверка отображения:
  * клиента (в разных форматах ФИО),
  * медкарты,
  * возраста,
  * телефона,
  * услуги,
  * ресурса,
  * комментариев,
  * меток пациента.

### **Настройки приёма и врача**
* Отображение фактического времени приёма.
* Настройки представления врача:
  * ФИО,
  * сортировка,
  * должность,
  * фотографии,
  * загруженность.

### **Цветовые настройки**
* Проверка выбора цветов:
  * статусов записи,
  * загрузки врачей.

### **Прочие настройки**
* Контроль занятости сотрудников.
* Отображение фильтров (горизонтально / вертикально).
* Настройка длительности приёма:
  * стандартные значения,
  * добавление нового значения,
  * удаление.

## **5️⃣ Сценарий «Проверка информации о враче»**
### **Проверяется:**
* Отображение информации о месте работы.
* Текстовые поля и декоративные элементы.
* Открытие карточки врача.
* Редактирование описания.
* Работа редактора текста:
  * шрифт,
  * размер,
  * выравнивание,
  * форматирование,
  * гиперссылки.
* Сохранение изменений.

## **6️⃣ Сценарий «Проверка создания записи»**
### **Цель:**
Полностью проверить процесс создания и обработки приёма.

### **Этапы:**
#### 🔹 Создание записи
* Переход в журнал записи под конкретного врача.
* Открытие окна **«Назначение приёма»**.
* Проверка всех подписей, полей и значений по умолчанию.

#### 🔹 Заполнение данных
* Пациент (с генерацией номера телефона).
* Причина записи.
* Длительность приёма (через кнопки и ручной ввод).
* Комментарий.
* Врач, ассистент, кабинет.
* Услуги и цены.
* Метки и статус записи.
* Выбор времени приёма.

#### 🔹 Завершение записи
* Проведение и закрытие документа.
* Повторное открытие записи из журнала.
* Проверка истории.
* Проверка печати документов.
* Начало приёма.

## **7️⃣ Проверка статусов приёма и контекстного меню**
### **Проверяются статусы:**
* Пациент ожидает.
* Приём начат.
* Приём окончен.
* Приём не состоялся.
* Подтверждён.
* Отменён.

### **Контекстное меню:**
* Начать приём.
* Создать план лечения.
* Создать наряд-заказ.
* Информация о пациенте.
* Отправка сообщения.
* Звонок пациенту.
* Добавление в лист ожидания.
* Печать талона.
* Обновление записи.
* Операции редактирования (вырезать).

## **8️⃣ Проверка отменённого приёма**
### **Выполняется:**
* Перевод приёма в статус **«Отменён»**.
* Фильтрация отменённых записей.
* Открытие отменённого приёма.
* Проверка всех полей, значений и статуса.
* Очистка фильтра.

## **9️⃣ Сценарий «Проверка старого журнала записи»**
### **Цель:**
Убедиться, что старый журнал записи работает корректно.

### **Проверки:**
* Переключение на старый журнал.
* Фильтры по неделям (5 недель).
* Фильтр по дням (7 дней).
* Выбор клиники.
* Фильтр отменённых записей.
* Работа выбора даты.
* Проверка печати.
* Проверка настроек.
* Интервалы графика.
* Цвета статусов и загрузки врачей.
