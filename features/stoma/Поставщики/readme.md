### Название теста
Создание поставщика (Контрагента типа «Поставщик») с договорами и проверкой всех данных

### Цель теста
Проверить полный процесс создания карточки поставщика в разделе «Запасы», включая:
- основные реквизиты,
- создание банковского счёта,
- создание трёх разных договоров,
- добавление нескольких видов контактной информации,
- проверку всех вкладок и корректность сохранённых данных (включая договоры действующие и архивные).

### Структура теста
Тест состоит из трёх основных сценариев:
1. Заполнение окна «Основное» (подготовка переменных)
2. Создание поставщика (основной сценарий)
3. Проверка созданного поставщика

### Что делает тест по порядку

**1. Подготовка**
- Закрывает все окна
- Удаляет все переменные
- Создаёт переменные с датами
- Генерирует случайное наименование поставщика (сохраняет в `$$Наименование$$`)

**2. Создание поставщика**
- Переходит в «Запасы» → «Контрагенты» → группу «Поставщики»
- Нажимает «Создать контрагента»
- Заполняет наименование и реестровый номер ОМС («START»)
- Переключает тип контрагента: сначала на «ИП», потом на «Юридическое лицо»
- Заполняет ИНН, КПП, код по ОКПО
- Сохраняет карточку
- Создаёт банковский счёт (через отдельный сценарий «Я создаю Банковский счёт Стомы») и выбирает его как основной
- Создаёт **три договора** с разными периодами действия:
  - Договор 1: действующий (дата начала в будущем, дата окончания позже)
  - Договор 2: архивный (даты в прошлом)
  - Договор 3: действующий (смешанные даты)
- Добавляет контакты:
  - Телефон (+7 987 …) с комментарием «Проверка» и попыткой позвонить
  - Email с комментарием «Проверка 2»
  - Юридический адрес (все поля «1»)
  - Фактический адрес (все поля «2»)
  - Почтовый адрес (все поля «3»)
  - Факс с комментарием «Проверка 3»
- Добавляет общий комментарий «Проверка комментария»
- Сохраняет и закрывает карточку поставщика

**3. Проверка созданного поставщика**
- Снова заходит в справочник Контрагенты
- Находит поставщика по наименованию через строку поиска
- Открывает карточку
- Переключается по всем основным вкладкам:
  - Банковские счета
  - Диалоги
  - История
  - История звонков
  - Источники
  - Наряд заказы
  - Программы страхования
  - Субъекты обращения МДЛП
  - Файлы
  - Финансы
  - Возвращается на «Основное»

**Проверка основных данных:**
- Наименование, ИНН, КПП, ОКПО, Реестровый номер ОМС
- Тип контрагента («Юр. лицо»)
- Банковский счёт
- Все контактные данные (телефон, email, 3 адреса, факс)
- Комментарии к контактам и общий комментарий
- Дата регистрации
- Родительская группа («Поставщики»)

**Проверка договоров:**
- Переходит в «Договоры действующие» и проверяет наличие договора №3
- Переходит в «Договоры архив» и проверяет наличие договоров №1 и №2
- Проверяет даты, вид цен и наименование договоров
