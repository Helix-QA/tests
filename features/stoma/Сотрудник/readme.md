### Общее назначение файла
**Функционал: Сотрудник**
Этот сценарий полностью тестирует создание и настройку карточки **сотрудника** (включая регистратуру), а также создание пользователя информационной базы с правами доступа.
Тест максимально покрывает всё, что связано с сотрудником: от заполнения основных данных до проверки всех вкладок, дополнительных настроек и создания логина для входа в программу.

### Структура сценария
**Сценарий: Создание карточки сотрудника**
#### 1. Подготовка переменных
Генерируются случайные уникальные данные, чтобы тест можно было запускать многократно без конфликтов:
- ФИО сотрудника (Фамилия + Имя + Отчество)
- Дата рождения, ИНН, СНИЛС
- Номер телефона и Email
- Паспортные данные (серия, номер, код подразделения, дата выдачи)

#### 2. Создание элемента (основная карточка сотрудника)
**Основная информация:**
- Заполняется ФИО, дата рождения, ИНН, СНИЛС

**Должность и роль:**
- Создаётся новая должность «Регистратура»
- Устанавливается роль «Регистратура»
- Настраивается отображение в журнале предварительной записи

**Контактная информация:**
- Добавляется телефон
- Добавляется Email
- Добавляется адрес (Москва)

**Удостоверение личности:**
- Добавляется паспорт гражданина РФ с заполнением всех полей

**Дополнительные разделы:**
- Добавляется специализация (просто открывается и закрывается окно)
- Добавляется кабинет
- Настраиваются параметры в правой колонке: вид цен, основной шаблон обследований, ассистент и т.д.

**Проверка всех вкладок карточки сотрудника:**
- График работы
- Услуги сотрудника
- Начисления и удержания
- Оценки и отзывы
- Настройки прав и доступа
- Промокоды
- Сегменты сотрудника
- Файлы

**Командная панель:**
- Проверка кнопок: настройка формы, история, дополнительные свойства, доп. сведения
- Сохранение карточки сотрудника

#### 3. Создание пользователя (Создание пользователя)
После создания карточки сотрудника тест переходит к созданию **пользователя информационной базы**:
- Находит только что созданного сотрудника в списке персонала
- Включает доступ к информационной базе («ДоступКИБ»)
- Генерирует уникальное имя для входа («РегистратураXXXX»)
- Назначает роль «Регистратура»
- Устанавливает период отборов администратора «За все время»
- Открывает расширенную настройку прав
- Проверяет дополнительные кнопки (настройка формы, история, доп. свойства)
- Сохраняет пользователя
В конце закрываются все окна.

### Ключевые особенности теста
- Используются только случайные данные — тест безопасно запускается много раз
- Полностью имитируется реальное создание сотрудника- регистратора
- Проверяются почти все вкладки и возможности карточки сотрудника
- Создаётся не только сотрудник, но и пользователь ИБ с правами
- Тест покрывает как основные реквизиты, так и дополнительные настройки (должность, контакты, удостоверение, права доступа)

### Краткое резюме для памяти
Этот файл **полностью тестирует создание сотрудника**:
1. Создаёт карточку сотрудника со всеми данными (ФИО, контакты, паспорт, должность «Регистратура» и т.д.).
2. Проверяет наличие и работу всех вкладок в карточке.
3. Создаёт пользователя информационной базы с логином и ролью «Регистратура».
Если через время откроешь файл, просто вспомни фразу:  
«Создаём полноценного сотрудника-регистратора + пользователя ИБ с правами доступа».
