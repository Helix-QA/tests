### Общее назначение файла
**Функционал: Финансы**
Этот сценарий проверяет работу всех основных финансовых отчётов в программе.  
Он создаёт разнообразные финансовые операции в клинике, а затем формирует отчёты и сравнивает их с заранее подготовленными эталонными макетами.

В файле проверяется **11 финансовых отчётов**.

### Структура сценариев
**Сценарий: Заполнение данными**  
Подготавливает окружение:  
- Удаляет старые переменные  
- Создаёт нового пациента  
- Создаёт нового врача  
- Создаёт товар и услугу  
- Создаёт нужные даты для дальнейшей работы  

**Сценарий: Создаю продажу с возвратом поциенту**  
Здесь происходит основная подготовка финансовых данных. Создаются операции разных типов:

- Продажа услуги наличными + возврат наличными пациенту  
- Пополнение лицевого счёта пациента  
- Прочие расходы (списание наличных на материалы)  
- Продажа услуги по банковской карте + возврат по карте  
- Продажа услуги за счёт лицевого счёта + возврат с лицевого счёта  
- Создание страхового полиса ОМС пациенту  
- Оказание услуги по страховому полису  
- Поступление денег от страховой компании  
- Возврат денег страховой компании  
- Закрытие кассовой смены (инкассация) и сдача наличных в банк  

**Сценарии проверки отчётов**
- **ОтчётОВыручке_ПроверкаОтчёта** — проверка отчёта «Отчёт о выручке»  
- **ВедомостьПоДенежнымСредствам_ПроверкаОтчёта** — проверка ведомости по денежным средствам  
- **ДаходыИРасходы_ПроверкаОтчёта** — проверка отчёта «Доходы и расходы»  
- **Затраты_ПроверкаОтчёта** — проверка отчёта по затратам (создаётся начисление зарплаты врачу)  
- **ВзаиморасчётыСПодотчётниками_ПроверкаОтчёта** — проверка взаиморасчётов с подотчётными лицами  
- **ОборотноСальдоваяВедомость_ПроверкаОтчёта** — проверка оборотно-сальдовой ведомости  
- **ШахматнаяВедомость_ПроверкаОтчёта** — проверка шахматной ведомости  
- **АнализОплатыПоВрачам_ПроверкаОтчёта** — анализ оплаты по врачам  
- **АнализОплатыПоПервичнымПостояннымИПотеряннымПациентам_ПроверкаОтчёта** — анализ оплаты по категориям пациентов  
- **ПрибылиИУбытки_ПроверкаОтчёта** — проверка отчёта «Прибыли и убытки»  
- **Прибыль_ПроверкаОтчёта** — проверка отчёта «Прибыль» (по номенклатуре)  
- **ОказаниеИОплаченныеУслуги_ПроверкаОтчёта** — оказанные и оплаченные услуги  
- **ОказанныеУслугиПродажиПоПоциентам_ПроверкаОтчёта** — оказанные услуги и продажи по пациентам  
- **ПланыЛечений_ПроверкаОтчёта** — проверка отчёта по планам лечений  
- **Реализации_ПроверкаОтчёта** — проверка отчёта «Реализации» (с использованием рекламации)  

### Как происходит проверка каждого отчёта
Для каждого отчёта выполняется одинаковая последовательность действий:
1. Переход по прямой ссылке на вариант отчёта  
2. Нажатие кнопки «НастроитьОтчет»  
3. Установка нужного фильтра (обычно по кассе «ОтчётСтома», врачу, пациенту или услуге)  
4. Формирование отчёта  
5. Сравнение полученного результата с эталонным макетом по шаблону  

### Ключевые особенности теста
- Тест максимально реалистично имитирует реальную финансовую жизнь клиники  
- Используются все основные способы оплаты: наличные, карта, лицевой счёт, страховка  
- Присутствуют как поступления, так и возвраты и расходы  
- Перед проверкой отчётов создаётся «живая» история движений денег  
- Все сравнения результатов идут **по шаблону**, чтобы тест не ломался из-за дат и номеров  

### Краткое резюме для памяти
Этот файл **создаёт полноценную финансовую активность** клиники (продажи, возвраты, страховку, расходы, инкассацию), а потом проверяет, что все ключевые финансовые отчёты формируются корректно и соответствуют ожидаемым макетам.
