## Загрузка данных

    Код начинает с подготовки тестовой среды: удаляются старые переменные и создаются основные объекты — пациент, врач, товары, услуги и даты. После этого формируются все необходимые переменные для работы с ними.

    Дальше идут различные сценарии работы с продажами и оплатами. Сначала создаются продажи наличными, затем производятся возвраты этих оплат. Аналогично обрабатываются пополнения лицевых счетов пациентов и прочие расходы. Платежи могут быть как наличными, так и картой, и для каждого сценария предусмотрен возврат. При этом код сохраняет ссылки на документы, чтобы потом быстро возвращаться к ним для проведения возвратов или повторных операций.

    Также реализованы операции со страховыми полисами: сначала создается полис для пациента, затем через него проводится продажа, фиксируется результат оказанной услуги, после чего поступления и списания денежных средств по полису обрабатываются через кассу. Код следит за привязкой к конкретным договорам и документам задолженности, чтобы корректно отражать все финансовые операции.

    Весь процесс строится как последовательность действий в интерфейсе клиентского приложения: переход по ссылкам, ввод данных, выбор из списков, проведение документов, обработка оплат и возвратов, закрытие окон. Сценарии повторяются для разных способов оплаты и типов операций, создавая полную тестовую среду, где можно проверять корректность работы всех финансовых и учетных процедур.
