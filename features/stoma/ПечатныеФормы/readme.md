### Общее назначение файла
**Функционал: Печатные формы**
Этот тест проверяет, что **все печатные формы** в программе правильно заполняются данными и соответствуют эталонным макетам.
Главная идея:  
Сначала подготавливается окружение (клиника и организация заполняются контактными и реквизитными данными), затем создаются нужные документы, и для каждого документа проверяется одна или несколько печатных форм. В конце данные очищаются, чтобы не засорять базу.

### Структура сценариев
**Сценарий: Загрузка данных**  
Подготовка окружения перед тестом:  
- Удаляются старые переменные  
- Загружаются основные данные (организация, структурная единица, даты)

**Сценарий: Заполнение данными клиники**  
Открывает карточку текущей структурной единицы (клиники) и заполняет её контактной информацией:  
телефон, email, полный адрес, индекс, координаты (широта/долгота), страна, валюта и т.д.  
Это нужно, чтобы в печатных формах корректно отображались реквизиты клиники.

**Сценарий: Заполнение данными организацию**  
Заполняет реквизиты текущей организации:  
ИНН, ОКПО, ОКАТО, ОГРН, тип (юридическое лицо) и снимает признак головной организации.

**Сценарий: Приём_**  
Создаёт документ **«Приём»** для пациента «Кулагина Надежда Тарасовна» с услугой «Чистка зубов» и сохраняет ссылку на него для дальнейших проверок.

Далее идут сценарии проверки печатных форм из документа **Приём**:
- **Приём_АктОВыполненныхРаботах**
- **Приём_ГарантийныйТалон**
- **Приём_КонсультацияВрача**
- **Приём_ТалонАмбулаторногоПациента**

Для каждой формы: открывается документ → нажимается «Печать» → выбирается нужная форма → сравнивается табличный документ с эталонным макетом по шаблону.

**Сценарий: Оплата**  
Выполняет оплату приёма наличными (с возможным закрытием кассовой смены) и делает возврат оплаты через помощник возврата.
После этого проверяются печатные формы, связанные с деньгами:
- **СписаниеДенег_АктОВозвратеДенежныхСредств**
- **ПоступлениеДенег_ПриходныйКассовыйОрдер**

**Сценарии проверки медицинских карт** (используется один и тот же пациент):
- **МедицинскиеКарты_АмбулаторнаяКарта043УА4**
- **МедицинскиеКарты_АмбулаторнаяКарта043УА5**
- **МедицинскиеКарты_МедицинскаяКарта025У04А5**
- **МедицинскиеКарты_МедицинскаяКарта025У14**
- **МедицинскиеКарты_МедицинскаяКарта025У25**
- **МедицинскиеКарты_АмбулаторнаяКарта043уСССРА4**
- **МедицинскиеКарты_АмбулаторнаяКарта043УСССРА5**

Для каждой: открывается медицинская карта пациента → меняется вид печатной формы → печатается → сравнивается с макетом.

**Сценарий: Поступление запасов и услуг**  
Создаёт документ поступления материалов (номенклатура «Обезбол») и сразу проверяет печатную форму **«Поступление запасов и услуг»**.

**Сценарий: Реализация**  
Создаёт документ **Реализация** на контрагента «ООО ГОССТОМА» с номенклатурой «Обезбол», проводит его и оплачивает. Сохраняет ссылку для дальнейших проверок.

Проверяются печатные формы из реализации:
- **Реализация_ТоварнаяНакладнаяТОРГ12**
- **Реализация_РасходнаяНакладная**
- **Реализация_АктОВыполненныхРаботах**
- **Реализация_УниверсальныйПередаточныйДокументУПД**

**Сценарий: СписаниеЗапасов**  
Создаёт документ списания запасов («Обезбол») и проверяет две формы:
- Требование-накладная (М-27)
- Акт о списании запасов

**Сценарий: ЗаказПоставщику**  
Создаёт заказ поставщику и проверяет печатную форму **«Заказ поставщику»**.

**Сценарий: ПланЛечения**  
Создаёт план лечения на 5 приёмов для пациента «Кулагина Надежда Тарасовна» и проверяет две формы:
- План лечения (медицинский)
- План лечения (финансовый)

**Сценарий: Очистка Клиники**  
Возвращает структурную единицу в исходное состояние — очищает все контактные данные, которые были заполнены в начале.

**Сценарий: Очистка Организации**  
Очищает реквизиты организации (ИНН, ОГРН и т.д.), возвращая её в исходное состояние.

### Ключевые особенности теста
- Перед проверками специально заполняются данные клиники и организации, чтобы печатные формы были «живыми» и содержали реальные реквизиты.
- В конце тест всё аккуратно очищает — база остаётся чистой.
- Все проверки идут **по шаблону** (`по шаблону`), чтобы небольшие расхождения в датах и номерах не ломали тест.
- Покрыты печатные формы из разных разделов: приёмы, оплаты, медицинские карты, складские документы, планы лечения.

### Краткое резюме для памяти
Этот файл **тестирует правильность заполнения всех важных печатных форм** в программе.  
Он заполняет реквизиты клиники и организации → создаёт документы → печатает формы → сравнивает с эталонами → и в конце всё очищает.
