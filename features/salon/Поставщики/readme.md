### Цель теста
Проверить, что поставщика можно успешно создать со всеми основными данными и контактной информацией, а после сохранения все введённые данные корректно отображаются.

### Из каких частей состоит тест
Тест состоит из трёх файлов:
- Основной сценарий создания поставщика
- Экспортный сценарий с переменными по времени и датам
- Экспортный сценарий создания банковского счёта

### Что делает основной сценарий (по порядку)

1. **Подготовка**
   - Запускает TestClient
   - Удаляет все старые переменные
   - Создаёт переменные с текущей датой и временем

2. **Создание поставщика**
   - Переходит в раздел Запасы → Поставщики
   - Нажимает кнопку Создать

3. **Заполнение основных данных**
   - Вводит случайное уникальное наименование поставщика
   - Включает и сразу выключает флаг «Комитент»
   - Переключает тип контрагента на «Юридическое лицо»
   - Заполняет ИНН, КПП и код по ОКПО
   - Выбирает вид цен «Розничная цена»

4. **Тег**
   - Создаёт новый тег с названием «Проверка»
   - Присваивает этот тег поставщику

5. **Банковский счёт**
   - Создаёт новый банковский счёт (через отдельный экспортный сценарий)
   - Выбирает созданный счёт как основной

6. **Контактная информация**
   - Добавляет телефон (начинается с +7 923 + случайные цифры)
   - Добавляет email (случайное число + @mail.ru)
   - Заполняет юридический адрес
   - Заполняет фактический адрес
   - Заполняет почтовый адрес
   - Добавляет комментарий «Проверка комментария»

7. **Дополнительные контакты**
   - Нажимает кнопку «Добавить контакт»
   - Добавляет ещё один телефон
   - Добавляет ещё один email
   - Добавляет факс

8. **Проверка вкладок**
   - Поочерёдно переключается между всеми вкладками карточки:
     - Банковские счета
     - Договоры
     - Задачи
     - История визитов
     - История звонков
     - Отчеты
     - Промокоды
     - Сообщения
     - Файлы
   - Возвращается обратно на вкладку «Основное»

9. **Сохранение**
   - Сохраняет карточку
   - Запоминает ссылку на созданного поставщика
   - Сохраняет и закрывает форму

### Сценарий проверки
После создания тест заново открывает карточку поставщика по сохранённой ссылке и проверяет, что все данные заполнены правильно:
- Наименование
- ИНН, КПП, ОКПО
- Вид цен
- Тип контрагента (Юр. лицо)
- Комитент (выключен)
- Комментарий
- Все контактные данные (телефон, email, три адреса, факс)
- Банковские реквизиты
- Тег
- Дата регистрации
- И другие элементы формы

### Важные моменты, которые использует тест
- Генерирует уникальное название поставщика каждый раз
- Использует случайные номера телефона и почты
- Создаёт тег «Проверка» прямо во время теста
- Создаёт банковский счёт с названием поставщика
- Заполняет адреса упрощённо (все поля «1», «2», «3»)
- Проверяет наличие всех вкладок в карточке поставщика

### Что проверяется в итоге
Что поставщик создаётся полностью, ничего не теряется при сохранении, все поля и вкладки работают корректно.
