Этот тест проверяет **основные CRM-объекты** конфигурации салона красоты / медцентра:  
- Заявки (лиды / потенциальные клиенты)  
- Задачи (встречи, звонки, претензии, прочее)  
- Сделки (воронка продаж)  
- Монитор визитов (дашборд с быстрыми переходами)

Тест написан с акцентом на **полное покрытие интерфейса**: создание, заполнение всех полей, переключение статусов/этапов, фильтры, кнопки действий, копирование/удаление, отображение в карточке клиента и в мониторе.

### Общий порядок и логика
1. **Подготовка данных и пользователей**  
   - Подключается TestClient под "Админ"  
   - Создаётся пользователь "Администратор салона красоты"  
   - Создаётся клиент, мастер, услуга, даты, переменные  
   - Переподключается под ролью "Администратор салона красоты" (чтобы проверить права именно администратора СК)

2. **Проверка справочника "Заявки"** (самый объёмный блок)
   - Переход в "CRM → Заявки"  
   - Создание новой заявки:  
     - Заполняются ФИО, телефон, email, адрес, соцсеть  
     - Канал привлечения ("Чат с клиентом")  
     - Рекламный источник ("Рекомендация клиента салона")  
     - Источник информации (ссылка на существующего клиента)  
     - Комментарий  
     - Дата  

   - Переключение статусов заявки:  
     Пропущенный звонок → Отвеченный звонок → Отвеченный диалог → Перезвонить → Некачественная → Обработана → "Записать на визит"  

   - Проверка фильтров и поиска:  
     Открытие фильтров по структурной единице, менеджеру, автору, источнику, пометке удаления  
     Поиск по клиенту → очистка строки поиска  

   - Действия над элементом:  
     Копирование → Удаление → Редактирование реквизитов  
     Переход по гиперссылкам статусов (Всего / Принята)

3. **Проверка справочника "Задачи"**
   Создаются 5 видов задач:  
   - Встреча (с клиентом)  
   - Звонок входящий (сразу старт выполнения)  
   - Звонок исходящий (на завтра)  
   - Прочее (с сотрудником, низкая важность, период)  
   - Претензия (высокая важность, по заявке, ответственный мастер)  

   Заполняются:  
   - Предмет (клиент / сотрудник / заявка)  
   - Тема, описание, взаимодействие  
   - Важность  
   - Сроки (дата / период)  

   Проверяются:  
   - Гиперссылки: Всего задач / Невыполненные / На сегодня / На завтра  
   - Копирование, редактирование, удаление

4. **Проверка формы "Сделки"**
   - Переход в "CRM → Сделки"  
   - Создание новой сделки:  
     - Сотрудник / менеджер  
     - Специализация  
     - Примечание, сумма  
     - Источник информации, канал привлечения  
     - Заявка (ссылка)  
     - Описание  
     - ФИО клиента + контактная информация (телефон, email, адрес, соцсеть)  

   - Действия в форме сделки:  
     Изменить вид сделки  
     Переключение этапов  
     Создание визита  
     Создание задачи  
     Создание заметки  
     Создание SMS / письма  
     Фильтры истории взаимодействий (все типы)  
     История объекта  
     Сохранение

5. **Отображение сделок в карточке клиента**  
   - Переход в карточку клиента  
   - Создание сделки из панели:  
     Первичный/повторный клиент  
     Постоянный клиент

6. **Проверка "Монитора визитов"** (дашборд CRM)
   - Переходы по всем гиперссылкам:  
     Все заявки → Все потенциальные клиенты → Все задачи → Сделки (две вкладки)  

   - Фильтры:  
     Выбор салона  
     Показать для всех сотрудников / только мои  
     Переключение периода (день / неделя / месяц)  

   - Быстрое создание:  
     Потенциальный клиент + первичный/повторный  
     Потенциальный + постоянный  
     Просто потенциальный  
     Сделка (первичный / постоянный)

7. **Финал**  
   - Переподключение обратно под "Админ"

### Кратко — что именно проверяется
- Полное создание и заполнение **заявки** (все поля + все статусы)  
- Создание всех видов **задач** (встреча, звонки, прочее, претензия)  
- Создание **сделки** со всеми реквизитами и взаимодействиями (визит, задача, заметка, SMS, письмо)  
- Фильтры, поиск, копирование, удаление, редактирование во всех справочниках  
- Отображение сделок в карточке клиента  
- Все быстрые действия и переходы в **Мониторе визитов**  
- Переключение между пользователями (Админ → Администратор СК → обратно)
