Этот тест проверяет **отчёты по финансам** в конфигурации салона красоты / медицинского центра.  
Главная цель — убедиться, что после разных операций с деньгами (продажи, возвраты, внесения, списания, инкассация, зарплата и т.д.) отчёты заполняются корректно и соответствуют ожидаемым макетам.

### Что происходит по порядку
1. **Подготовка данных**  
   Создаётся клиент, мастер, товар, пакет услуг, даты.  
   Запоминаются текущие организация и структурная единица.  
   Снимается контроль остатков на складе (чтобы не мешал проводить документы).

2. **Создание разных видов поступлений и списаний денег**  
   - Покупка обычной услуги/товара с полной оплатой наличными через кассу  
   - Прочие поступления наличными (100,50)  
   - Прочие списания наличными (10,50) по статье «Возврат аванса клиенту»  
   - Внесение на лицевой счёт клиента (500,00)  
   - Продажа в долг → оплата частично наличными (131,00), остаток в долг  
   - Полный возврат оплаты по предыдущей продаже (с возвратом наличными)  
   - Инкассация (выемка) из кассы 15,50

3. **Начисление зарплаты мастеру**  
   Создаётся вид начисления «Оклад»  
   Начисляется 200 руб. мастеру  
   Документ проводится

4. **Закрытие кассовой смены**  
   В нескольких местах теста происходит закрытие смены (Z-отчёт) — это важно, потому что многие отчёты по выручке и ОФД завязаны на закрытые смены.

5. **Проверка самих отчётов** (самая важная часть теста)
   - **Отчёт по выручке**  
     Фильтр по кассе «ОтчётФинансы»  
     Проверяется соответствие макету «МакетОтчётПоВыручке»

   - **Ведомость по денежным средствам**  
     Фильтр по кассе «ОтчётФинансы»  
     Проверяется макет «МакетОтчётВедомостьПоДенежнымСредствам»

   - **Прибыль и убытки — по сотруднику**  
     Фильтр по конкретному мастеру  
     Проверяется макет «МакетОтчётПрибыльИУбыткиСотрудник»

   - **Прибыль и убытки — по кассе**  
     Фильтр по кассе «ОтчётФинансы»  
     Проверяется макет «МакетОтчётПрибыльИУбыткиКасса»

   - **Доходы и расходы**  
     Фильтр по кассе «ОтчётФинансы»  
     Проверяется макет «МакетОтчётДоходыИРасходы»

   - **Анализ оплаты по новым / постоянным / потерянным клиентам**  
     Фильтр по кассе «ОтчётФинансы»  
     Проверяется макет «МакетАнализОплатыПоНовымПостояннымИПотеряннымКлиентам»

6. **Финальные действия**  
   - Продажа пакета услуг (абонемента) — чтобы проверить, как пакеты влияют на отчёты  
   - Закрытие смены ещё раз  
   - Выключение вспомогательных настроек:  
     снимается галка «Авансовая схема пробития услуг» с организации  
     возвращается контроль остатков на складе

### Кратко — что именно проверяется
Тест имитирует типичный «финансовый день» салона и смотрит, правильно ли считаются и отображаются:

- выручка по кассе (включая продажи, возвраты, внесения на лицевой счёт, инкассацию)  
- прочие поступления и списания  
- влияние зарплаты на прибыль  
- движение денег по кассе  
- доходы и расходы  
- структура клиентской базы по оплатам (новые, постоянные, потерянные)

Всё это с фильтром по одной тестовой кассе «ОтчётФинансы», чтобы результат был предсказуемым и не зависел от других операций в базе.

### Главные особенности теста
- Очень много операций с наличными и кассой  
- Многократное закрытие смены (Z-отчёт) — это критично для отчётов по выручке и ОФД  
- Проверяется влияние пакетов услуг и лицевых счетов  
- Зарплата учитывается в отчёте «Прибыль и убытки»  
- Все проверки — через сравнение табличного документа с эталонным макетом

«Тест создаёт разные движения денег по кассе, начисляет зарплату, закрывает смены и проверяет, что шесть ключевых финансовых отчётов заполняются правильно при фильтре по тестовой кассе».
