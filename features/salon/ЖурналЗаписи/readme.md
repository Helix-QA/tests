Этот тест проверяет **функциональность и настройки нового (современного) журнала записи** в конфигурации салона красоты / медцентра.  
Главная цель — убедиться, что:
- журнал записи открывается  
- все настройки отображения, фильтров, цветов, интервалов, горячих клавиш работают без ошибок  
- контекстное меню визита содержит все нужные команды и они выполняются (или хотя бы открываются)  
- переключение между новым и старым журналом записи возможно

Тест очень большой и состоит из двух частей:  
1. **Новый журнал записи** — основной сценарий с кучей настроек и проверок  
2. **Старый журнал записи** — короткий сценарий переключения туда-обратно

### Что происходит по порядку (основной сценарий — Новый журнал записи)
1. **Подготовка данных**  
   - Создаётся мастер (Мастер_3)  
   - У мастера задаётся график работы: циклический, с 05:00 до 23:30 каждый день  
   - Создаётся услуга салона, клиент, даты

2. **Переход в журнал записи + открытие настроек**  
   - Открывается «Журнал записи» из главного меню  
   - Нажимается кнопка настроек (ОткрытьФормуНастройки_0)  
   - Если появляется предложение вернуться к новому журналу — подтверждается

3. **Проверка и переключение всех вариантов отображения клиента**  
   Прогоняются все возможные способы показа имени клиента в визите:  
   - Не отображать  
   - ФИО полностью  
   - Имя Отчество  
   - Только Имя  
   - Фамилия И.О.  
   - Фамилия Имя  
   - Имя Фамилия  
   (тест просто переключает и смотрит, что ничего не падает)

4. **Настройки отображения визита**  
   Включаются/выключаются галочки:  
   - Отображение ресурса (сотрудника)  
   - Отображение услуги  
   - Отображение телефона  
   - Отображение комментария визита  
   - Отображение комментария о клиенте

5. **Настройки отображения сотрудника**  
   Переключаются варианты представления сотрудника:  
   - Представление  
   - Имя  
   - Фамилия И.О.  
   - Фамилия Имя  
   - ФИО  
   Сортировка: по ФИО / по должности  
   Галочки: должность, загруженность, фото, ассистент

6. **Настройки цветов**  
   Открываются палитры для изменения цвета фона:  
   - Статусы визита: Ожидается, Онлайн-запись, Подтверждён, Пришёл, Не пришёл, Окончен, Окончен не оплачен  
   - Загрузка сотрудников: Низкая, Средняя, Высокая, Максимальная  
   (тест просто открывает и закрывает окна выбора цвета)

7. **Настройки контроля записи**  
   Переключаются галочки:  
   - Не контролировать занятость мастера  
   - Контролировать одновременные услуги  
   (включают/выключают, чтобы проверить, что интерфейс не ломается)

8. **Настройки интервалов времени**  
   Устанавливаются разные шаги сетки: 5, 10, 30, 60, 15 минут

9. **Расположение фильтров**  
   Переключается между вертикальным и горизонтальным отображением фильтров

10. **Горячие клавиши**  
    - Открываются настройки горячих клавиш  
    - Создаётся новая горячая клавиша (выбираются разные действия: визит, журнал, продажа и т.д.)  
    - Сохраняется (если появляется предупреждение — подтверждается)

11. **Переключатели периода и фильтры**  
    - Переключение дня назад/вперёд  
    - Календарь выбора даты  
    - Переключение День / Неделя  
    - Открытие фильтров: Продажа, Клиент, Статус, Подтверждён, Теги, Статус клиента  
    - Смена салона (структурной единицы)  
    - Печать предварительной записи

12. **Фильтры вертикального журнала**  
    - Переключение отображения сотрудников  
    - Выбор конкретного мастера из дерева

13. **Фильтр по времени суток**  
    - Переключение: Утро / Вечер / Весь день

14. **Создание тестового визита**  
    - Клик по свободному слоту → «Запланировать визит»  
    - Заполняется: мастер, клиент, услуга, время 17:00, длительность 2 часа  
    - Сохраняется

15. **Проверка контекстного меню визита** (самая важная часть — правый клик по визиту)  
    Прогоняются почти все пункты меню:  
    - Подтвердить визит  
    - Клиент пришёл  
    - Визит окончен  
    - Оплатить (с кассовой оплатой + закрытие смены, если нужно)  
    - Копировать визит  
    - Вырезать визит  
    - Добавить в лист ожидания  
    - Визиты клиента  
    - О клиенте (открывается карточка)  
    - Отправить сообщение клиенту (ожидается ошибка — нет телефона)  
    - Установить нерабочее время / рабочее время / зарезервировать время  
    - Добавить в лист ожидания (с заполнением клиента и дат)

### Короткая вторая часть — Старый журнал записи
- Переход в настройки → выбор «Вернуться к старому журналу записи»  
- Открытие журнала → проверка, что открылся старый  
- Возврат к новому журналу записи

### Кратко — что именно проверяется
- Все настройки отображения (имя клиента, сотрудника, услуги, комментарии, фото, должность и т.д.)  
- Цвета статусов и загрузки  
- Интервалы сетки времени  
- Расположение фильтров  
- Горячие клавиши  
- Фильтры по периоду, сотруднику, времени суток, салону  
- Создание визита  
- **Полное контекстное меню визита** — все действия из правого клика работают или хотя бы открываются без падений

Если одним предложением:  
«Тест полностью прогоняет интерфейс нового журнала записи: все настройки отображения, фильтры, цвета, горячие клавиши + создание визита и проверка абсолютно всего контекстного меню визита (подтверждение, оплата, копирование, лист ожидания, нерабочее время и т.д.)».
