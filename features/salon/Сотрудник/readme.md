Этот тест проверяет **полный цикл работы со справочником "Сотрудники"** в конфигурации салона красоты / медцентра.  
Главная цель — убедиться, что **все вкладки, кнопки, поля и подчинённые объекты сотрудника работают корректно**, ничего не падает при создании, заполнении и переключении.

Тест написан как **"прогон интерфейса"** — создаётся один сотрудник со случайным ФИО, заполняются почти все возможные реквизиты, вкладки, графики, документы, карты, файлы и т.д., и проверяется, что каждая кнопка/переключатель открывается или сохраняет данные без ошибок.

### Что происходит по порядку
1. **Подготовка данных**  
   - Генерируется случайное число → формируется уникальное ФИО сотрудника (ФамилияХХХ, ИмяХХХ, ОтчествоХХХ)  
   - Генерируется случайный промокод (для вкладки Промокоды)  
   - Переменные запоминаются

2. **Создание сотрудника**  
   - Открывается справочник "Персонал → Сотрудники"  
   - Нажимается "Создать"

3. **Заполнение основной информации**  
   - ФИО (случайные)  
   - Роль пользователя → "Руководитель"  
   - Должность: ищется "Администратор", если нет — создаётся

4. **Контакты** (закомментировано в коде, но структура есть)  
   - Телефон, город, email (в коде закомментировано, но логика присутствует)

5. **Документы (удостоверения личности)**  
   Создаются и заполняются 4 вида документов:  
   - Паспорт (серия, номер, дата выдачи/окончания, кем выдан)  
   - Водительское удостоверение  
   - Личная медицинская книжка  
   - Патент на трудовую деятельность  

6. **Карты**  
   - Добавляется две карты (магнитная + сгенерированный код)  
   - Нажимается "Сгенерировать код карты"

7. **Специализация**  
   - Добавляется специализация "ОООДДД" (если нет — создаётся)

8. **Дополнительные реквизиты**  
   - Дата рождения  
   - ИНН  
   - Страховой номер ПФР  
   - Фото (загружается из файла Фото2.png через контекстное меню)  
   - Комментарий "Тест"  
   - Доступные кассы: включается флаг "Выгружать" для "Основная касса"

9. **Вкладка "График работы"**  
   Прогоняются все три режима заполнения графика:  
   - **Разово** → интервал 10:00–12:00  
   - **По дням недели** → интервал 13:00–17:00  
   - **Циклически** → интервал 12:00–13:00  
   Каждый раз открывается мастер заполнения → добавляется интервал → сохраняется → формируется график

10. **Вкладка "Услуги сотрудника"**  
    Переключаются все фильтры:  
    - Только выбранные  
    - По специализации  
    - Все

11. **Вкладка "Начисления и удержания"**  
    - Фильтр "Только активные" / "Все"  
    - Нажимается "Создать" (и сразу закрывается)

12. **Вкладка "Оценки и отзывы"**  
    Просто переход (без действий)

13. **Вкладка "Отчеты"**  
    Просто переход

14. **Вкладка "Промокоды"**  
    - Поиск по сгенерированному коду  
    - Если промокода нет — создаётся новый

15. **Вкладка "Присоединённые файлы"**  
    - Загружаются два файла (Фото2.png и Фото1.jpg)  
    - Один помечается на удаление  
    - Создаётся копия  
    - Поиск по слову "фото" → очистка поиска

16. **Финал**  
    - Возврат на вкладку "Основное"  
    - Сохранение и закрытие сотрудника  
    - Закрытие списка сотрудников

### Кратко — что именно проверяется
- Создание сотрудника со случайным ФИО  
- Заполнение **всех основных полей** (ФИО, роль, должность, контакты, ИНН, ПФР, дата рождения, фото, комментарий)  
- Добавление **всех видов удостоверений личности** (паспорт, права, медкнижка, патент)  
- Карты доступа (генерация кода)  
- Специализации  
- Доступные кассы  
- **Все режимы графика работы** (разово, по дням, циклически)  
- Фильтры во вкладках "Услуги", "Начисления", "Промокоды"  
- Загрузка и управление присоединёнными файлами  
- Переход по **всем вкладкам** карточки сотрудника без падений

Если одним предложением:  
«Тест создаёт сотрудника со случайными данными, заполняет абсолютно все возможные поля, документы, графики, карты, файлы и вкладки, прогоняет все переключатели и кнопки — чтобы убедиться, что карточка сотрудника полностью работоспособна».
