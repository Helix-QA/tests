## 1. Блок «Продажи»

В рамках автотеста создаётся документ «Справка в налоговую» и выполняется проверка всех доступных вариантов печатных форм.

Проверяются:

* Справка в налоговую по форме КНД 1151160 (ФОУ)
* Справка в налоговую по форме КНД 1151158 (ОУ)
* Справка в налоговую по форме КНД 1151156 (МУ)

Для каждой формы:

* выполняется формирование печатной формы,
* производится сравнение табличного документа с эталонным макетом,
* дополнительно контролируется наличие QR-кода (для регламентированных форм).

## 2. Блок «Финансы»

Создаются документы поступления и списания денежных средств. После этого проверяются печатные формы кассовых документов.

Проверяются:

* Приходный кассовый ордер
* Расходный кассовый ордер (форма КО-2)
* Акт о возврате денежных сумм (форма КМ-3)

Для каждой формы выполняется формирование печати и сравнение с шаблоном.

## 3. Блок «Персонал»

Моделируется полный цикл начисления заработной платы с созданием тестовых данных (сотрудник, клиент, номенклатура).

Проверяются печатные формы документа «Начисление зарплаты»:

* Платёжная ведомость
* Расчётные листки
* Расшифровка начисления
* Расшифровка начисления (подробно)

Каждая форма формируется после расчёта и проведения документа и сравнивается с эталонным макетом.

## 4. Блок «Запасы»

Проверяется широкий перечень складских документов и их печатных форм.

Проверяются:

* Заказ поставщику
* Поступление товаров:

  * Приходный ордер (М-4)
  * Приходная накладная
* Возврат поставщику
* Инвентаризация:

  * Инвентаризация
  * ИНВ-19
  * ИНВ-3
  * ИНВ-3 (без фактических данных)
* Акт об оприходовании товаров
* Акт списания запасов
* Акт списания с себестоимостью
* Перемещение запасов:

  * Накладная
  * Накладная по типу цен
* Отчёт комитенту о продажах

По каждому документу формируется печатная форма и выполняется сравнение с эталоном.

## 5. Отчёт «Расшифровка начислений и удержаний»

Проверяется корректность формирования отчёта системы компоновки данных.

Выполняется:

* открытие варианта отчёта,
* установка отбора по сотруднику,
* формирование отчёта,
* сравнение результата с эталонным макетом.

## 6. Очистка тестовых данных

В завершение автотеста выполняется возврат системы в исходное состояние.

Производится:

* очистка реквизитов организации (ИНН, КПП, ОГРН),
* очистка данных структурной единицы,
* снятие установленных флагов в помещениях.

### Итог

Автотест представляет собой комплексный регрессионный сценарий, охватывающий:

* продажи,
* финансы,
* персонал,
* складской учёт,
* расчёт заработной платы,
* отчётность.

Основной механизм проверки — сравнение сформированного табличного документа с эталонным макетом. Это позволяет контролировать корректность печатных форм после изменений в конфигурации и обеспечивает стабильность функциональности системы.
