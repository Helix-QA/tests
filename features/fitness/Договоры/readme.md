### Общая цель теста
Проверить, что создание договора в 1С с разными типами бланков (HTML, внешний файл .docx, встроенный редактор) проходит без ошибок, все ключевые поля заполняются, формы открываются, кнопки работают.

### Последовательность выполнения теста (по сценариям)
1. **Подготовка тестовых данных в справочниках**  
   - Заполняются «мусорными» значениями (123, тестовые строки) почти все важные реквизиты **Организации**:  
     - КПП, ОКПО, ОГРН, код налогового органа  
     - Банковский счёт по умолчанию  
     - Ставка НДС (создаётся новая «НДС покупателю» 1%)  
     - Все ответственные лица (руководитель, гл. бухгалтер, кассир, кладовщик) → ФИО и должности  
     - Контактная информация: телефон, email, адрес (с индексом, домом и т.д.), факс  
   - Заполняются «мусорными» значениями **Структурная единица**:  
     - Телефон, email  
     - Полный адрес  
     - Геокоординаты (широта, долгота)  
     - Загружается тестовый логотип (jpg-файл)

2. **Загрузка внешнего файла в бланк договоров**  
   - Открывается справочник «Бланки договоров»  
   - Находится строка «Внешний файл»  
   - Очищается старый файл (если был)  
   - Загружается файл **Договор.docx** из папки features  
   - Сохраняется

3. **Создание и проверка договора с типом «HTML шаблон»**  
   - Открывается список «Договоры» → Создать  
   - Выбирается бланк «HTML шаблон»  
   - Заполняются поля:  
     - Префикс номера → «123»  
     - Клиент → поиск и выбор «Станислав Стас Стасиевич»  
     - Членство/пакет услуг → того же клиента  
     - Реализация → поиск по номеру «1234567»  
   - Переход во вкладку «Дополнительно»  
   - Нажимается кнопка «Отправить» → проверяется, что открывается форма исходящего письма  
   - Пробуются действия с подписью:  
     - Разрешить редактирование реквизитов  
     - Подключить подпись по SMS (открытие формы)  
     - Установка и снятие флага «Подписан»  
   - Сохранение и закрытие договора

4. **Создание и проверка договора с типом «Внешний файл»**  
   - То же самое, что в п.3, но:  
     - Выбирается бланк «Внешний файл»  
     - Дополнительно выбирается поле «Основной договор» (открывается список договоров → закрывается)  
     - Остальные шаги идентичны (клиент, членство, реализация, «Отправить», подпись)

5. **Создание и проверка договора с типом «Встроенный редактор»**  
   - То же самое, что в п.3, но:  
     - Выбирается бланк «Встроенный редактор»  
     - Дважды нажимается «Отправить» (видимо, особенность формы или проверка на повторное открытие)  
     - Установка и снятие флага «Подписан»

6. **Очистка тестовых данных (возврат в исходное состояние)**  
   - Открывается Организация → очищаются все ранее заполненные поля:  
     - Реквизиты (КПП, ОКПО и т.д.) → пусто  
     - Ставка НДС → «Без НДС»  
     - Ответственные лица → пусто  
     - Контакты, адреса → пусто  
   - Открывается Структурная единица → очищаются:  
     - Контакты, адрес, геокоординаты → пусто  
     - Удаляется загруженная картинка логотипа  
   - Все окна закрываются

### Что тест проверяет на самом деле
- Интерфейс не падает при заполнении и выборе разных типов бланков  
- Поля клиента, членства, реализации находятся и выбираются  
- Кнопка «Отправить» открывает форму письма  
- Флаг «Подписан» можно ставить/снимать  
- Форма подписи по SMS открывается  
- Все три типа бланков поддерживаются и проходят базовый happy path

### Что тест НЕ проверяет
- Правильность подстановки данных в сам текст договора  
- Реальное формирование pdf / docx  
- Отправку письма  
- Реальную подпись  
- Наличие ошибок валидации  
- Содержимое сформированного документа

Если коротко запомнить одной фразой:  
«Тест гоняет создание договора по трём типам бланков с одним и тем же клиентом и реализацией, проверяет заполнение и базовые кнопки, потом чистит за собой организацию и СЕ».
