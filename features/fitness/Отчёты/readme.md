#### Описание автотеста проверки Отчёты

## Загрузка данных

    Сценарий начинается с подготовки тестовой среды: открывается клиентское приложение или подключается уже существующее, после чего удаляются все старые переменные и загружаются активные пакеты услуг и членства. Затем создаются и проводятся документы по реализации услуг и членств, чтобы система имела актуальные данные для отчётов.

    Далее создаются тестовые объекты: два клиента, два тренера, несколько помещений для персональных и групповых занятий, а также соответствующие услуги. Создаются переменные для организации и структурной единицы, чтобы всё корректно связывалось в системе. После этого загружаются продажи и оплаты, проводятся документы реализации, чтобы в системе появилось движение денежных средств.

    После подготовки данных выполняется проверка различных отчётов. Для отчёта «Активные пакеты услуг» открывается соответствующий вариант отчёта, настраивается фильтр по нужной номенклатуре пакета услуг и формируется таблица для сравнения с макетом. Аналогично проверяются отчёты «Активные членства», «Продажи и оплаты», «Персональные занятия», «Групповые занятия» и «Посещения» — в каждом случае настраиваются фильтры, формируются отчёты и сравниваются с шаблонами макетов.

    Также создаются различные финансовые операции: поступления на счёт, оплату услуг наличными и картой, возвраты, списания денежных средств и инкассация. Всё это позволяет проверить корректность формирования отчётов по выручке и оплатам.

    В итоге код создаёт полную тестовую среду с клиентами, мастерами, услугами и операциями, проводит все необходимые документы и проверяет корректность всех отчётов по шаблонам, чтобы можно было быть уверенным в правильной работе системы.
