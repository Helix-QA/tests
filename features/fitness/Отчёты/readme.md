# 📌 Общая цель функционала

**Функционал:** Отчёты
**Роль:** Тестировщик
**Цель:** Проверить корректность формирования отчётов и вывода данных на основании подготовленных документов и операций.

Сценарий покрывает:

* Подготовку тестовых данных
* Продажи и оплаты
* Персональные и групповые занятия
* Посещения
* Движение денежных средств
* Проверку выручки
* Сравнение табличных документов с макетами

---

# 🔹 1. Подготовительный этап — Загрузка данных

В сценарии **«Загрузка данных»** выполняется:

### 📂 Очистка и подготовка

* Удаление переменных
* Загрузка активных пакетов услуг
* Загрузка активных членств
* Проведение документов:

  * Реализация
  * ЧленствоПакетУслуг

### 👥 Создание сущностей

Создаются:

* 2 клиента
* 2 тренера
* 2 помещения
* Персональная услуга
* Групповая услуга
* Организация и структурная единица

### 💰 Продажи и оплаты

* Загружаются продажи
* Проводятся документы реализации

📌 **Назначение этапа:**
Создать корректную тестовую базу данных, чтобы отчёты имели реальные данные для формирования.

---

# 🔹 2. Проверка отчётов по активным данным

## 📊 Отчёт «Активные пакеты услуг»

Проверяется:

* Поиск варианта отчёта
* Отключение диаграммы
* Настройка отбора:

  * Поле: *Номенклатура пакета услуг*
  * Значение: **Фитнес Старт**
* Установка периода
* Формирование отчёта
* Сравнение табличного документа с макетом:

```
МакетАктивныеПакетыУслуг
```

📌 Проверяется корректность отображения активных пакетов на заданную дату.

---

## 📊 Отчёт «Активные членства»

Аналогичная логика:

* Поиск отчёта
* Настройка отбора:

  * Поле: *Номенклатура членства*
  * Значение: **Фитнес Безлимит**
* Формирование
* Сравнение с:

```
МакетАктивныеЧленства
```

📌 Проверяется корректность отображения активных членств.

---

## 📊 Отчёт «Продажи и оплаты»

Проверяется:

* Добавление отбора по полю **Комментарий**
* Значение: *Продажа и оплата*
* Установка периода 01.01.2025 – 31.12.2025
* Формирование
* Сравнение с:

```
МакетПродажиИОплаты
```

📌 Проверяется корректность отражения продаж и оплат за период.

---

# 🔹 3. Проверка отчётов по занятиям

## 🧍 Персональные занятия

### Подготовка:

* Создаётся продажа персональной услуги
* Проводится документ
* Производится оплата
* Создаётся занятие
* Статус: **Выполнено**

### Проверка:

* Отчёт «Персональные занятия»
* Отбор по сотруднику (тренеру)
* Сравнение с:

```
МакетПерсональныеЗанятия
```

📌 Проверяется, что выполненные персональные занятия корректно попадают в отчёт.

---

## 👥 Групповые занятия

### Подготовка:

* Продажа групповой услуги
* Создание группового занятия
* Добавление клиента в состав
* Статус: **Выполнено**

### Проверка:

* Отчёт «Групповые занятия»
* Отбор по клиенту
* Сравнение с:

```
МакетГрупповыеЗанятия
```

📌 Проверяется корректность отражения групповых занятий.

---

## 🏃 Отчёт «Посещения»

* Отбор по клиенту
* Формирование
* Сравнение с:

```
МакетПосещения
```

📌 Проверяется корректность отображения визитов клиента.

---

# 🔹 4. Проверка движения денежных средств

Сценарий включает:

### 💵 Операции:

* Поступление наличных
* Оплата через кассу

* Возврат клиенту
* Оплата банковской картой

* Списание денежных средств
* Инкассация

📌 Это формирует полноценное движение денежных средств по кассе "Проверка отчётов".

# 🔹 5. Отчёт по выручке

В финале проверяется:

* Установка параметра:

  * Касса = **Проверка отчётов**
* Формирование отчёта
* Сравнение с:

```
МакетПоВыручке
```

📌 Проверяется:

* Корректность расчёта выручки
* Учёт поступлений
* Учёт возвратов
* Учёт списаний
* Учёт инкассации
