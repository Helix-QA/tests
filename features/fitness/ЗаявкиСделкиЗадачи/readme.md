# Проверка и создание ЗаявкиСделкиЗадачи

Для автотестирования созданы следующие фича-файлы:

- **`ЗаявкиСделкиЗадачи`**

Для автотестирования созданы следующе макеты:
- **`ЗаявкиСделкиЗадачи`**
- **`Export_ЗаявкиСделкиЗадачи`**

#### Описание автотеста проверки ЗаявкиСделкиЗадачи

    Этот сценарий предназначен для комплексного тестирования работы CRM-системы, включая справочники «Заявки», «Сделки» и «Задачи». Тест охватывает создание данных, проверку интерфейса, фильтров, поиска и основных операций с объектами.

    1. Подготовка тестовой среды:
    Сценарий сначала закрывает все окна клиентского приложения, создаёт администратора и под его учётной записью — руководителя отдела продаж. После этого создаются базовые объекты для работы: клиент, тренер, услуга, членство, даты, сделки и продажи. Все переменные и вспомогательные функции инициализируются, а сеанс администратора закрывается.

    2. Авторизация под Руководителем отдела продаж:
    Проверяется возможность входа и корректного закрытия информационных окон, чтобы убедиться, что пользователь видит чистый интерфейс и может полноценно работать.

    3. Работа со справочником «Заявки»:
    Сценарий имитирует создание заявок, включая ввод контактной информации, выбор рекламных источников, статусов звонков, добавление заметок и сообщений, проверку фильтров и поиск. Также реализована работа с повторным созданием заявок, созданием сделок на основе заявок и проверка всех фильтров в списке заявок.

    4. Работа со справочником «Сделки»:
    Проверяются действия с видами сделок, каналами привлечения, рекламными источниками, менеджерами, этапами сделки. Заполняются обязательные поля: примечания, суммы, исполнители, описание, интересы и теги. Сценарий проверяет фильтры, изменение видов сделок и проведение оплаты.

    5. Работа со справочником «Задачи»:
    Тест охватывает создание задач, установку различных периодов и фильтров, назначение исполнителей, даты выполнения и типа взаимодействия. Реализована проверка переключателей, режимов напоминаний, флагов выполнения взаимодействия и корректность поиска по задачам.

    6. Проверка связей и данных клиента:
    Проверяется корректность отображения сделок в карточке клиента, привязка сделок к членствам и возможность проведения сделок напрямую через интерфейс клиента.

    7. Завершение работы:
    Сценарий завершает сеанс Руководителя отдела продаж и возвращается к администратору, чтобы подготовить систему к следующему циклу тестирования.

    Итог:
    Сценарий обеспечивает полный функциональный цикл тестирования CRM: создание и редактирование заявок, сделок и задач, проверку фильтров, поиска и статусов, а также корректность привязки данных к клиентам и членствам. Позволяет убедиться, что все базовые операции работают корректно, данные сохраняются и интерфейс адекватно реагирует на действия пользователя.
