**Описание этого автотеста (Заявки → Сделки → Задачи в CRM фитнес-клуба)**
Этот тест — **длинный smoke / happy path + проверка интерфейса и фильтров** для трёх основных объектов CRM:  
Заявки → Сделки → Задачи, плюс проверка связей с клиентом и членством.

### Что происходит по порядку (основные этапы)
1. **Подготовка данных (Сценарий: Загрузка данных)**  
   - Подключается под Админом  
   - Создаётся пользователь «Руководитель отдела продаж»  
   - Создаётся тестовый набор данных:  
     - Клиент (с переменными: Фамилия, Имя, Отчество, телефон, email и т.д.)  
     - Тренер  
     - Групповая услуга  
     - Членство  
     - Даты (вероятно рабочие периоды)  
     - Сделка  
     - Продажа членства  
   - Все эти сущности сохраняются и запоминаются в переменные для дальнейшего использования

2. **Вход под пользователем «Руководитель отдела продаж»**  
   - Подключается TestClient от имени этого менеджера  
   - Закрываются все окна и возможное окно «Информация»

3. **Работа с Заявками (самый объёмный сценарий)**  
   - Переход в список «Заявки»  
   - Создаётся **новая заявка** от неизвестного клиента:  
     - Заполняются ФИО, телефон, email, адрес, соцсеть, мессенджер  
     - Менеджер = текущий пользователь (Руководитель отдела продаж)  
     - Рекламная площадка = «Ловец лидов»  
     - Создаётся и выбирается новый рекламный источник «Пробная»  
     - Комментарий, дата = сегодня  
   - Проверяются **все статусы заявки** (кнопки нажимаются по очереди):  
     Пропущенный звонок → Отвеченный звонок → Отвеченный диалог → Перезвонить → Некачественная → Обработана  
   - Создаётся заметка, сообщение, взаимодействие  
   - Статус «Обработана» → перевод в потенциального клиента + новая продажа  
   - Проверяются **все фильтры** в меню фильтрации (активируются по одному):  
     Менеджер, Сотрудник регистрации, Рекламный источник, Канал привлечения, Период создания, Дата обработки, Пометка удаления, Оценка, Внешняя система и др.  
   - Проверяется **поиск** по фамилии  
   - Создаётся **вторая заявка** — уже по существующему клиенту  
     - Выбирается существующий клиент  
     - Создаётся сделка (Новая продажа + Продление)  

4. **Работа со Сделками**  
   - Переход в список «Сделки»  
   - Проверяется массовая обработка: «Изменить» → разные действия (вид сделки, канал, источник, менеджер)  
   - Проверяются быстрые фильтры: просроченные, на сегодня, без активности  
   - Проверяются **все фильтры** в меню (очень много): периоды, статус, этап, менеджер, тег, причина отказа, сегмент и др.  
   - Меняются виды сделки (ЗадачиСделкиЗанятия ↔ Новая продажа)  
   - Открывается существующая сделка по клиенту  
   - Заполняются дополнительные поля:  
     - Примечание, сумма сделки, менеджер презентации (тренер), флаг презентации  
     - Описание  
     - Добавляются тег и интерес  
   - Проверяются фильтры взаимодействий (заметки, сообщения)  
   - Нажимаются этапы сделки (1,2,3)  
   - Завершается сделка успешно:  
     - Результат = Успешно  
     - Создаётся реализация → добавляется групповая услуга → оплата  

5. **Работа с Задачами**  
   - Переход в список «Задачи»  
   - Проверяются быстрые периоды (4 дня, неделя, месяц, день)  
   - Проверяются **все фильтры** (статус, вид взаимодействия, вид сделки, этап, контрагент, исполнитель и др.)  
   - Проверяется массовая обработка «Изменить» (исполнитель, даты, вид взаимодействия)  
   - Создаётся **новая задача**:  
     - По существующему клиенту  
     - Тема и описание  
     - Срок / период  
     - Напоминание → не напоминать  
     - Флаги взаимодействия (вкл/выкл)  
     - Исполнитель  
   - Проверяется поиск по теме задачи  

6. **Проверка связей в карточке клиента и членства**  
   - Открывается карточка созданного клиента  
   - Проверяется вкладка сделок:  
     - Открытие сделки  
     - Смена вида сделки (на ЗадачиСделкиЗанятия)  
   - Открывается созданное членство  
   - Проверяется кнопка создания сделки → Продление → проведение  

7. **Выход**  
   - Подключается обратно Админ

### Главное, что проверяет этот тест
- Создание и базовое заполнение **Заявок, Сделок, Задач** под конкретным менеджером  
- Работоспособность **всех статусов, этапов, кнопок завершения**  
- Корректность **массовых действий «Изменить»**  
- Полнота и доступность **всех фильтров** в списках (очень много проверок именно фильтров)  
- Связанность объектов: заявка → клиент → сделка → задача → членство → реализация  
- Отсутствие падений интерфейса при нажатии всех этих кнопок и переходах  
- Поиск и выбор существующих элементов работает  

### Короткая памятка для себя
«Тест создаёт тестового клиента + тренера + членство + услугу, заходит под РОП, создаёт заявку → переводит в сделку → завершает сделку с оплатой → создаёт задачу → проверяет ТЫСЯЧУ фильтров и кнопок «Изменить» → смотрит связи в карточке клиента и членства → выходит под админа».
