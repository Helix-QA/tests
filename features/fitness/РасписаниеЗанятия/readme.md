# Общая цель
* Представляет собой автотест (Gherkin) для проверки **«Расписания занятий» в 1С**
* Проверяет:
  * интерфейс
  * создание занятий
  * фильтры
  * бизнес-логику

# Подготовка данных
* Запускается тест-клиент
* Создаются:
  * помещение
  * помещение расписания
  * сотрудник
  * клиент
* Создаются услуги:
  * персональная
  * групповая
  * аренда зала
* Создаются переменные (дата)
* Выполняются продажи клиенту
Используется как база для всех тестов

# Проверка интерфейса
* Переход: **Персонал → Расписание занятий**
* Проверяются:
  * переключатели:
    * помещения
    * сотрудники
    * по графику / все
  * фильтры (все виды)
  * мастер заполнения
  * печать
  * настройки
  * режимы отображения:
    * день / неделя
    * утро / вечер / весь день

# Персональные занятия
* Устанавливается фильтр: персональные занятия
* Выбирается сотрудник
* Создаётся занятие через клик по расписанию
* Заполняются:
  * услуга
  * помещение
  * клиент
  * длительность (150)
  
  Проверяется:
* дата
* клиент
* тип = персональное
* сотрудник
* услуга
* основание оплаты

# Групповые занятия
* Устанавливается фильтр: групповые занятия
* Выбирается помещение
* Создаётся занятие
* Дополнительно:
  * добавляется клиент в состав
  * настраивается занятие

  Проверяется:
* услуга
* помещение
* длительность
* наличие клиента
* элементы интерфейса

# Аренда зала
* Устанавливается фильтр: аренда зала
* Выбирается помещение
* Создаётся занятие
* Заполняются:
  * клиент (арендатор)
  * услуга
  * длительность

  Проверяется:
* клиент
* услуга
* основание оплаты
* длительность
* таблица состава (статус клиента)

# ⏱ Резерв времени
* Устанавливается фильтр: резерв
* Создаётся резерв
* Заполняются:
  * длительность (180)
  * комментарий
  * цвет

  Проверяется:
* комментарий
* длительность
* помещение

# Технические особенности
* Используются клики по координатам (работа с сеткой расписания)
* Применяются переменные:
  * $$Клиент$$
  * $$Услуга$$
* Используются проверки по шаблону (для дат и документов)

# Коротко (чтобы вспомнить за 5 сек)
* Подготовка данных
* Проверка интерфейса
* 4 типа:
  * персональные
  * групповые
  * аренда
  * резерв
* Везде:
  создать → заполнить → проверить
