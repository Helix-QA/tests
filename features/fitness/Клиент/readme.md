**Описание этого автотеста (Проверка заполнения карточки клиента + статусы + объединение)**
Это один из самых объёмных и важных тестов по клиентской части системы фитнес-клуба.  
Он проверяет **почти всю карточку клиента** от создания до объединения дубликатов, а также **логику изменения статусов** клиента в зависимости от продаж и возвратов.

### Основные блоки теста (по сценариям)
1. **Сценарий 01. Создание клиента**  
   Создаётся полностью заполненная карточка **нового клиента** со случайными данными.

   Что именно заполняется:
   - ФИО (случайные «ФамилияXXXX», «ИмяXXXX», «ОтчествоXXXX»)
   - ИНН (случайный корректный)
   - Дата рождения
   - Интересы и теги (создаются новые и отмечаются)
   - Все виды контактной информации:
     - Телефон
     - Email
     - Адрес
     - Соцсеть
   - Паспорт (серия, номер, дата выдачи, код подразделения, гражданство «Россия»)
   - Договор (просто добавляется пустой)
   - Карта (генерируется код карты)
   - Родственник (дочь, с созданием нового клиента-родственника)
   - Автомобиль (марка + номер)
   - Медицинская справка (новая с наименованием и сроком)
   - Канал и источник маркетинга
   - Персональный менеджер отдела сервиса
   - Заметка рецепции

   Дополнительно проверяются / открываются почти все кнопки и вкладки карточки:
   - Фото (проверка сообщения о веб-камере)
   - Взаимодействия (встреча, заметка)
   - Пробный пакет, занятие, продажа членства/пакета
   - Финансы, продажи, посещения, занятия, взаиморасчёты, достижения
   - E-mail, SMS, договоры, задачи, история, звонки, отзывы, файлы

   Особый блок — **медицинское обследование**:
   - Берутся все измерения из шаблона «Мед обследование»
   - Для каждого измерения создаётся запись в регистре значений (вес, давление, % жира и т.д.)
   - Значение ставится тестовое «2000»

2. **Сценарий 02. Статус клиента**  
   Проверяется **автоматическое изменение статусов** клиента в зависимости от действий.

   Последовательность статусов:
   - Создаётся чистый клиент → статус **«Потенциальный член клуба»**
   - Продаётся услуга → статус **«Клиент клуба»**
   - Продаётся членство → статус **«Член клуба»**
   - Делается возврат членства + выполняется регламентное задание → статус **«Бывший член клуба»**
   - Продаётся новый пакет услуг → статус обратно **«Клиент клуба»**

   Дополнительно:
   - Создаётся потенциальный клиент
   - Продаётся пробное членство + пакет услуг → статус остаётся **«Потенциальный»**
   - Создаётся сделка со статусом «Отказ» → проверяется статус **«Отказ»**

3. **Сценарий: Возврат настройки периода поиска бывших клиентов**  
   Восстанавливается исходное значение настройки «N дней до бывшего клиента»

4. **Сценарий: Я создаю клиента для объединения**  
   Проверяется **функция объединения дубликатов клиентов**.

   Что происходит:
   - Создаётся клиент 1 (только ФИО + дата рождения + встреча)
   - Создаётся клиент 2 (тот же ФИО + дата + телефон + адрес + карта + автомобиль)
   - Открывается клиент 1 → кнопка «Объединение клиентов»
   - Находит дубли → отмечает одного как оригинал
   - Выполняет замену ссылок
   - После объединения открывается объединённый клиент и проверяются:
     - Телефон
     - Адрес
     - Карта
     - Автомобиль
     - Пол
     - ФИО

### Что тест проверяет в целом
- Полноту заполнения **всех** основных и дополнительных реквизитов карточки клиента
- Корректность работы форм добавления: КИ, документы, родственники, авто, медсправки, карты, интересы/теги, маркетинг
- Работоспособность **всех вкладок и кнопок** в карточке клиента (очень много переходов)
- Логику **автоматического изменения статусов** клиента (потенциальный → клиент → член → бывший → клиент)
- Обработку **продаж, возвратов, пробных пакетов**
- Функцию **объединения дубликатов** с сохранением данных (КИ, карты, авто и т.д.)
- Отсутствие падений интерфейса при большом количестве действий

### Короткая памятка для памяти
«Самый большой тест по клиенту: создаёт клиента со всем-всем (КИ, паспорт, авто, родственник, мед.обследование, карта, теги…), проверяет ВСЕ вкладки, потом гоняет статусы по цепочке Продажа → Возврат → Продажа → Отказ, в конце объединяет двух одинаковых клиентов и проверяет, что данные слились правильно».
