### Цель теста
Проверить полный процесс создания карточки поставщика, заполнение всех основных и дополнительных реквизитов, контактной информации, банковского счёта, переключение между вкладками и корректность сохранения данных.

### Структура теста
Тест состоит из трёх основных частей:
- Контекст (запуск TestClient)
- Сценарий создания поставщика
- Сценарий проверки созданного поставщика

### Что делает сценарий «Создание поставщика»

1. **Переход к созданию**
   - Переходит в справочник Контрагенты
   - Выбирает группу «Поставщики»
   - Нажимает кнопку «Создать»

2. **Основные реквизиты**
   - Генерирует случайное уникальное наименование поставщика
   - Включает и сразу выключает флаг «Комитент»
   - Переключает тип контрагента: сначала на «Физическое лицо / ИП», затем на «Юридическое лицо»

3. **Дополнительные юридические данные**
   - Заполняет ИНН, КПП, ОГРН, код по ОКПО
   - Заполняет ФИО директора организации
   - Заполняет ФИО контактного лица
   - Нажимает кнопку «Записать»

4. **Банковский счёт**
   - Создаёт новый банковский счёт (через отдельный экспортный сценарий «Я создаю Банковский счёт Фитнес»)
   - Выбирает его как основной банковский счёт

5. **Контактная информация**
   - Добавляет основной телефон (+7 987 … случайные цифры)
   - Добавляет основной email (случайное число + @mail.ru)
   - Заполняет юридический адрес (все поля «1»)
   - Заполняет фактический адрес (все поля «2»)
   - Заполняет почтовый адрес (все поля «3»)
   - Добавляет комментарий «Проверка комментария»

6. **Переключение между вкладками**
   - Поочерёдно открывает все доступные вкладки карточки:
     - E-mail
     - Банковские счета
     - Данные диалогов чат-бота
     - Диалоги
     - Задачи
     - История
     - История звонков
     - Продажи
     - Состав сегментов
     - Файлы (два раза)
     - Финансы
   - Возвращается на вкладку «Основное»

7. **Добавление дополнительных контактов**
   - Добавляет второй телефон
   - Добавляет второй email
   - Добавляет факс
   - Сохраняет карточку
   - Запоминает ссылку на созданного поставщика

### Сценарий «Проверка созданного поставщика»

1. **Открытие карточки**
   - Переходит по сохранённой ссылке к созданному поставщику
   - Создаёт переменные с текущей датой

2. **Работа с тегами**
   - Открывает меню тегов
   - Пытается добавить тег «Проверка»

3. **Проверка заполнения данных**
   Тест проверяет, что после сохранения корректно отображаются:
   - Наименование поставщика
   - ИНН, КПП, ОГРН, ОКПО
   - ФИО директора и контактного лица
   - Тип контрагента («Юр. лицо»)
   - Флаг «Комитент» выключен
   - Комментарий
   - Основной банковский счёт
   - Все виды контактной информации (телефоны, email, три адреса, факс)
   - Дата регистрации (по шаблону текущей даты)
   - Все декорации и заглушки на форме
